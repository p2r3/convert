This is a fork. convert-api wraps the p2r3/convert browser-based converter with an Express REST API, headless Chromium, native fast-paths via sharp, and integrations for yt-dlp, Tesseract OCR, and Whisper transcription. All upstream WASM handlers are still available — see /api/formats.

convert-api

REST API for file/URL conversion. Sync below the 100ms threshold, async (with job IDs + ETA) above it. See /docs for the interactive Swagger UI, /openapi.json for the spec.

The 100ms rule

Every request is estimated up front. If the work is expected to take more than 100ms, the response is HTTP 202 with: 

{
  "jobId": "8b1f8e6a-...",
  "kind": "screenshot",
  "status": "queued",
  "estimateMs": 6000,
  "estimatedSeconds": 6.0,
  "statusUrl":  "http://.../api/jobs/8b1f.../",
  "resultUrl":  "http://.../api/jobs/8b1f.../result",
  "streamUrl":  "http://.../api/jobs/8b1f.../stream",
  "pollAfterMs": 3000
}

Then poll statusUrl, stream streamUrl, or long-poll resultUrl?wait=true. Force inline with ?sync=true; force async with ?async=true.

Quick examples

URL → PNG

curl -X POST http://localhost:3000/api/screenshot \
  -H 'content-type: application/json' \
  -d '{"url":"https://example.com","format":"png","fullPage":true}' \
  -o example.png

YouTube → MP3

curl -X POST http://localhost:3000/api/ytdlp \
  -H 'content-type: application/json' \
  -d '{"url":"https://www.youtube.com/watch?v=...","format":"mp3"}' \
  -o song.mp3

Image upload → WebP (sharp fast-path, inline)

curl -X POST http://localhost:3000/api/convert \
  -F file=@photo.jpg -F to=webp -F quality=80 \
  -o photo.webp

Subscribe to new conversions (webhook)

curl -X POST http://localhost:3000/api/subscriptions/formats \
  -H 'content-type: application/json' \
  -d '{"url":"https://you.example/hook","secret":"…"}'
