convert-api

REST API on Express for file/URL conversion. Spins up headless Chromium on demand.

Endpoints

GET /health

Liveness check. Returns whether the browser converter dist/ is built.

GET /api/formats

Lists supported formats. Query params: ?category=image, ?direction=to|from.

POST GET /api/screenshot

Render a URL to PNG / JPEG / WEBP / PDF. Detects YouTube and waits for the player.

curl -X POST http://localhost:3000/api/screenshot \
  -H 'content-type: application/json' \
  -d '{"url":"https://example.com","format":"png","fullPage":true}' \
  -o example.png

YouTube example (renders the embedded player frame):

curl -G 'http://localhost:3000/api/screenshot' \
  --data-urlencode 'url=https://www.youtube.com/watch?v=dQw4w9WgXcQ' \
  -o yt.png

Or grab the thumbnail directly (no browser needed):

curl -G 'http://localhost:3000/api/screenshot' \
  --data-urlencode 'url=https://www.youtube.com/watch?v=dQw4w9WgXcQ' \
  --data-urlencode 'youtubeThumbnail=true' \
  -o yt-thumb.jpg

POST /api/convert

Convert an input (uploaded file or a remote URL) to a chosen output format.

curl -X POST http://localhost:3000/api/convert \
  -F file=@photo.jpg -F to=webp -F quality=80 \
  -o photo.webp

URL → PNG (works for any website; YouTube uses the player frame):

curl -X POST http://localhost:3000/api/convert \
  -H 'content-type: application/json' \
  -d '{"url":"https://example.com","to":"png"}' \
  -o example.png

URL pointing to a file (downloads then converts):

curl -X POST http://localhost:3000/api/convert \
  -H 'content-type: application/json' \
  -d '{"url":"https://upload.wikimedia.org/wikipedia/commons/4/47/PNG_transparency_demonstration_1.png","to":"jpeg","quality":85}' \
  -o demo.jpg

Conversion engine layers

  1. URL → image/PDF: Puppeteer directly.
  2. Image → image: sharp native fast-path (PNG, JPEG, WEBP, AVIF, TIFF, GIF, HEIF).
  3. Everything else: Drives the existing browser-based converter via Puppeteer (requires dist/ build).
Tip: build the full converter once with bun run build (or npm run build) to unlock the 90+ format handlers behind /api/convert.
